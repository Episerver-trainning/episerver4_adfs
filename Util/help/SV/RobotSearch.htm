Hantera webbadresser

Adressen till webbplatsen kan visas i adressfältet på tre olika sätt med EPiServer. Den kan vara helt dynamisk och återge den unika sidans id-nummer. Den dynamiska adressen kan också anpassas så att sökmotorer kan indexera webbplatsen. Det är också möjligt att visa adressen utifrån strukturen på webbplatsen. Detta kan styras genom att redaktören lägger upp namn för varje sida eller att det styrs av namnet på sidan.

De flesta sökmotorer på internet söker av webbplatser efter information och lagrar informationen i sina databaser. Man brukar kalla det att indexera webbplatser. De flesta sökmotorer klarar inte av att indexera webbplatsen där adressen innehåller frågetecken (?). Som standard innehåller alla adresser till sidor i EPiServer ett frågetecken och likamed ett unikt nummer. För att sökmotorer ska kunna indexera webbplatsen med automatik är det därför viktigt att vi ställer om adresserna så sökmotorerna kan indexera sidan. Om du vill att sökmotorer på internet ska kunna indexera er webbplats markera alternativet Använd webbadresser som möjliggör för sökmotorer att indexera webbplatsen och välj Spara. Om du tittar på adressen till en specifik sida har EPiServer gjort om adressen från www.foretaget.se/templates/page.aspx?id=1234 till www.foretaget.se/templates/page____1234.aspx. Det gör att sökmotorerna kan indexera sidan.

Om du vill att adressen ska visas utifrån strukturen så aktiverar du det genom att markera alternativet Använd webbadresser som återspeglar webbplatsens struktur och välj Spara. Om du har uppgraderat en tidigare version av EPiServer till 4.60 är detta alternativ inte aktivt. För att aktivera det måste du ändra en inställning i web.config. Läs mer om det i dokumentet web.config settings.

Remote Web Sites

The Remote Web sites function is to enable to you to retrieve information between the various EPiServer Web sites. This is used for load balanced Web sites and content mirroring, amongst other things. To be able to take information from one Web site to another, is a function that is often requested by customers with several Web sites on EPiServer on several different machines, such as a public Web site and an intranet, etc. Editors need only create information in one place, but it can be displayed in all.

With the Remote Web sites function, you create the connections to the Web sites that you want to be able to send information to and receive information from. In admin mode it is possible to set the function completely. A number of changes in web.config are also required. The changes required depends on which functions you are going to use it for. You can read more about this under Load Balancing and Clustering in EPiServer and Content Mirroring Configuration.

More information on this can be found at www.episerver.com.

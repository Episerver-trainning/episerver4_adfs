WSRP Sources

WSRP (Web Service for Remote Portlets) is a technology and standard that means that it is possible to consume and produce information and services that can be used by others without actually needing technical platforms.

At the core of the technology, you build so-called Portlets, which produce various types of information or services. Applications/services (so-called producer) can have several portlets that produce contents and services from its own application. It is the producer that primarily decides what can be consumed by someone else.

EPiServer supports the producer and the consumer. This means that you can allow others to use information from your Web site through portlets created in EPiServer. In a standard installation of EPiServer there are examples of producing page contents, news, etc. With the help of the EPiServer Portal Framework, it is also possible to consume information from other WSRP sources. Read more about Portals and WSRP in EPiServer Portals.

To be able to consume portlets in EPiServer, you must have EPiServer Portal Framework. In a standard installation there is a demo example of it. Contact your supplier if you want to use EPiServer Portal Framework in scrape operation. To be able to consume portlets in EPiServer, you also need to add the various sources that will be available in admin mode.

Add a new WSRP source as follows:

  1. Select WSRP sources on the Config tab.
  2. Select Register new. If you want to change an existing source, click on the name.
  3. Enter a name for the source in the Name field. This is an arbitrary name that will be shown to users of the portal.
  4. You can either choose to provide the source through WSDP or via Web services. If you choose Register WSDL, provide the source in a field. If you choose Register Web services , you must provide addresses to the source in four fields. You can obtain these details from the WSRP service provider.
  5. Select Save.

By choosing the Test button, you can test the source.

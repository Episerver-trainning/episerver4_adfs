Form Settings

The dialog box Edit form is used for creating/editing a form. Here, you can build up a table and add the different form fields as you want. You can also select the properties each field is to have.

  1. In the Name of form field, you select a name for the form. If you are going to send the form by e–mail, the name will be shown in the Subject row on the e-mail message being sent. This name is also displayed in the list of available forms, so you can use this form on another page.
  2. In the Directory for the form field, you choose which directory the form should be placed in.
  3. The Form can be sent without logging on field should be selected if you want anyone on your Web site to be able to send in the form. If it is not checked, the visitor must be logged on in order to reply.
  4. The Same person can send the form several times field should be selected if you want the same person (computer user) to be able to send the form several times. If this option is left empty, visitors will only be able to send in the form once per computer. This works by placing a cookie on the visitor's computer.
  5. In the Thank you page shown upon response field, you can point to a page that is to be shown when a visitor submits a response. Create the page on the Ordinary Web page page template and then select the page as a thank you page.

Building up the form

In the lower part of Edit form you create a table with the buttons below the table. Add row is always placed at the bottom of the table. Add column is always placed furthest right in the table. If you select Insert row you can insert a row above the row that is currently selected. Insert column opens a column to the left of the selected column. Select a row or column by clicking on an inserted field in the cell. If there are no fields in the table, you can start by dragging in a field.

Position the various fields by grabbing the field in question at the right edge and dragging it to the required position in the table. Each cell in the table can only contain one form field. When you have dragged a form field into the cell, the settings for the relevant field type will be opened to the right of the table. Depending on the field you have selected, you need to make various settings. Some properties exist in all types of field, and some in all input fields. These properties exist in all fields:

CSS class

is a field where you can define a CSS class (format) for what the field should look like. The class must be present in the CSS file used for the Web site. Contact your system administrator for more information.

Name

is the field where you define a name for the field, which enables you to know which field the visitor has filled in. The simplest thing is to use the same or similar name to the one you used in the heading field which belongs to the field. You can only use A-Z, a-z, 1-9 and _. Spaces cannot be used.

Heading

can be created in a field of its own, or you can write it in conjunction with the input field. If you write it in conjunction with the input field, the heading will be displayed to the left of the field. This alternative should be selected in order to make the form highly accessible. A heading is the text to be displayed to visitors so that they know what to fill in.

Screen tips

is a field where you can give visitors more information about what to fill in. The text in the screen tip is displayed when you move the mouse over the input field.

Value must be entered

is a field that means that you are forcing the visitor to enter a value in the field. If the visitor leaves the field empty, an error message is displayed saying that the field must be filled in. The error message refers first to the text you have specified in the heading field. If you do not have any heading, it refers to whatever you have entered in the Name field.

Validate as

is a field that enables you to check whether the visitor has filled in the right type of information in a field. You can check whether it is an e-mail address, check various date formats and various types of numbers.

When all the properties for each field have been completed, the button Save will be shown under the properties. You must save each individual field and its settings. If you want to delete a field, select the Delete button under the field's properties.

Heading

Heading is the field type used to let visitors know what they should enter in each field.

  • In the Text field, enter the text which is to be shown to visitors.

Text field

Text field is the field type which gives visitors an input field in the form of a row in which they can enter text. This is used for short text information you want the visitor to enter.

  • In the Width field, enter how wide the field is to be in terms of characters.
  • The Value must be entered field should be selected if you do not want visitors to be able to send in the form without filling in this particular field.

Button

Use of this button to give visitors a button to press when they want to send in the form.

  • In the Button text field, enter the text to be displayed on the button.
  • In the Result from posting field, select what will happen when the visitor sends in the form. You can select to save it in a database and/or send it by e-mail. You can also specify that the form should be sent to a Web address where you can have a separate application dealing with the information. Depending on the option you choose, different fields are shown.
  • In the Send e-mail to this address field, state the e-mail address that a message should be sent to when a visitor posts a form.
  • In the E-mail sender field, state the sender’s address that should be in the message sent.
  • In the E-mail subject field, state the heading that should be shown in the subject box in the message that is sent when a visitor sends a form.

Text area

Text area is the field type which gives visitors a large area in which to type in text. You can decide the width of the area in terms of characters and how many rows the field should have.

  • In the Width field, enter how wide the field is to be in terms of characters.
  • In the Height row, specify the number of rows that the text area should have.

Select box

The Select box is the field type which gives visitors a pull-down menu in which they can select one of several alternatives which you have specified.

  • In the Options field, enter the various options which the visitors are to choose from. Type in the options and separate them by pressing ENTER between each option.

Radio button

The Radio button is the field type which gives visitors a list of options in which they can select one of several alternatives which you have specified.

  • In the Placementfield, select whether you want the option to be shown horizontally or vertically.
  • In the Options field, enter the various options which the visitors are to choose from. Type in the options and separate them by pressing ENTER between each option.

Checkbox

Checkbox is the field type which gives visitors a list of various options, from which they can select one of several alternatives that you have specified.

  • In the Placementfield, select whether you want the option to be shown horizontally or vertically.
  • In the Options field, enter the various options which the visitors are to choose from. Type in the options and separate them by pressing ENTER between each option.

When the form is complete and all fields and their properties have been saved, save the entire form with the Save button in the middle of the Edit form dialog box. Save and publish the page.

The Form Data tab

If you chose to save the information in the database in your form, you can find all the forms which have been submitted under the Form data tab. This tab is displayed next to the Version list when you have saved and published your form page.

This is what you do to retrieve data from the database:

  1. Select the Form data tab and a search form will be shown.
  2. Select which form you want to get data from. Usually there is only one form to select.
  3. Enter two dates if you want to specify your search by time. These fields can be left empty if you want to retrieve all the postings made.
  4. Select option Display data for all pages if you want to get data from all pages using the current form.
  5. Enter how many hits you want per page.
  6. Select Search.

When you have searched for the submitted forms, they will be listed below the search form. You can browse amongst pages if there are many of them, and you can also choose to delete all visible contributions from here.

If you want to process the information in Microsoft Excel, select the Export to Excel button. To do this, Excel must be installed on your computer. A software component must also be installed on your computer. You can read more about this under Personalize EPiServer in the Editor's manual.
